## 1. 项目概述

### 1.1 研究目标

本项目实现了 **SE-CCM（Surrogate-Enhanced Convergent Cross Mapping）** 方法，用于在耦合混沌网络系统中进行统计严格的因果推断。核心思路是：

1. 通过 **CCM 算法** 检测变量间的因果关系（基于 Sugihara 等人 2012 年提出的收敛交叉映射）
2. 结合 **5 种替代数据方法**（FFT / AAFT / iAAFT / 时移 / 随机重排）构建零假设分布
3. 通过假设检验（p 值 + FDR 校正）判断因果关系的统计显著性
4. 在多种混沌系统（Lorenz / Logistic / Hénon）和网络拓扑（ER / WS / Ring）上系统评估检测性能

### 1.2 主要功能

| 功能模块 | 说明 |
|---------|------|
| 动力系统生成 | 耦合 Logistic 映射、Lorenz 系统、Hénon 映射网络 |
| 网络拓扑 | Erdős-Rényi (ER)、Watts-Strogatz (WS)、环形 (Ring) |
| CCM 算法 | 延迟嵌入、参数自动选择（MI + FNN）、KDTree 最近邻 |
| 替代数据 | FFT、AAFT、iAAFT、时移、随机重排 共 5 种方法 |
| 统计检验 | 秩 p 值、z 分数、Benjamini-Hochberg FDR 校正 |
| 性能评估 | TPR、FPR、Precision、F1、AUC-ROC、AUC-PR、Cohen's d |
| 可视化 | 收敛曲线、因果矩阵热图、网络拓扑图、替代数据分布图 |
| 批量实验 | 双变量验证、耦合强度扫描、噪声扫描、拓扑对比实验 |

## 4. 核心概念与约定

### 4.1 邻接矩阵约定

```
A[i, j] = 1  表示  j → i  （j 驱动 i，j 是因，i 是果）
```

- 行索引 `i` = 接收者（效果端）
- 列索引 `j` = 发送者（原因端）
- 这意味着 `A.sum(axis=1)` 给出每个节点的入度（被多少节点驱动）

**示例**：3 节点网络，节点 0→1，节点 0→2
```python
adj = np.array([
    [0, 0, 0],   # 节点 0：没有输入
    [1, 0, 0],   # 节点 1：接收来自 0 的输入
    [1, 0, 0],   # 节点 2：接收来自 0 的输入
])
```

### 4.2 CCM 约定

```python
ccm(x, y, E, tau)
```

- 从 x 的影子流形 M_x 交叉映射预测 y
- 检验的假设是 **"y 导致 x"**（y causes x）
- 返回值 ρ：预测值与真实 y 之间的 Pearson 相关系数

**直觉解释**：如果 y 确实驱动了 x，那么 x 的吸引子重构中会包含 y 的信息，从 M_x 可以准确预测 y。

### 4.3 替代数据约定

当检验 "j → i" 这条因果边时：

- **替代化的目标是原因变量 j**（而非效果变量 i）
- 逻辑：如果将 j 的时间结构打乱（生成替代数据），而 CCM 值显著下降，说明 j 对 i 的信息确实有意义

### 4.4 嵌入参数

- **τ（时间延迟）**：通过自互信息（Auto Mutual Information）的第一个极小值选取
- **E（嵌入维度）**：通过假近邻法（False Nearest Neighbors, FNN）选取
- 每个节点独立选取参数（非全局统一）

## 5. 配置文件详解

配置文件位于 `configs/default.yaml`，以下逐项说明：

### 5.1 全局参数

```yaml
seed: 42                    # 随机种子基数（实际种子 = seed + 重复序号）
```

### 5.2 时间序列参数

```yaml
time_series:
  T: 3000                   # 生成的有效时间步数（去除暂态后）
  transient: 1000           # 暂态步数（前 1000 步丢弃，等系统到达吸引子）
```

> **建议**：T ≥ 2000 以保证 CCM 收敛；暂态步数 ≥ 500

### 5.3 动力系统参数

```yaml
logistic:
  r: 3.9                    # Logistic 参数（3.9 处于混沌区间）

lorenz:
  sigma: 10.0               # Lorenz σ 参数
  rho: 28.0                 # Lorenz ρ 参数
  beta: 2.6667              # Lorenz β = 8/3
  dt: 0.01                  # 积分步长（ODE 采样间隔）

henon:
  a: 1.4                    # Hénon a 参数
  b: 0.3                    # Hénon b 参数
```

### 5.4 CCM 参数

```yaml
ccm:
  E_max: 10                 # FNN 搜索的最大嵌入维度
  tau_max: 50               # MI 搜索的最大时间延迟
  k: null                   # 最近邻数（null = E+1，Sugihara 原始设定）
  n_library_sizes: 20       # 收敛曲线的库大小采样点数
```

### 5.5 替代数据参数

```yaml
surrogate:
  methods:                   # 可用方法列表
    - fft                    # FFT 相位随机化
    - aaft                   # 幅度调整傅里叶变换
    - iaaft                  # 迭代 AAFT（推荐，精度最高）
    - timeshift              # 循环时移
    - random_reorder         # 随机重排（最激进）
  n_surrogates: 100          # 每次检验生成的替代数据数量
  iaaft_max_iter: 200        # iAAFT 最大迭代次数
```

> **方法选择建议**：
> - `iaaft`：最推荐，同时保持幅度分布和功率谱
> - `aaft`：较快，近似保持两者
> - `fft`：仅保持功率谱，适用于高斯近似的数据
> - `timeshift`：保持局部结构，仅破坏耦合同步
> - `random_reorder`：最激进，破坏所有时间结构

### 5.6 统计检验参数

```yaml
testing:
  alpha: 0.05               # 显著性水平
  fdr_method: bh             # Benjamini-Hochberg FDR 校正
```

### 6.1 动力系统生成器 (`generators/`)

#### 时间序列生成

```python
from surrogate_ccm.generators import LogisticNetwork, LorenzNetwork, HenonNetwork

# Logistic 映射网络
system = LogisticNetwork(adj, coupling=0.1, r=3.9)
data = system.generate(T=3000, transient=1000, seed=42, noise_std=0.0)
# data.shape = (3000, N)

# Lorenz 系统网络（连续 ODE，RK45 积分）

data = system.generate(T=3000, transient=1000, seed=42)

# Hénon 映射网络
system = HenonNetwork(adj, coupling=0.05, a=1.4, b=0.3)
data = system.generate(T=3000, transient=1000, seed=42)
```

**工厂函数**（通过名称创建系统）：

```python
from surrogate_ccm.generators import create_system

system = create_system("logistic", adj, coupling=0.1)
data = system.generate(T=3000, transient=1000, seed=42)
```

#### 各系统动力学方程

**Logistic 映射**：
```
X_i(t+1) = (1 - ε) · r · X_i(t) · (1 - X_i(t))
          + ε · Σ_j A[i,j] · r · X_j(t) · (1 - X_j(t)) / k_in_i
```

**Lorenz 系统**：
```
dx_i/dt = σ(y_i - x_i) + ε · Σ_j A[i,j] · (x_j - x_i)
dy_i/dt = x_i(ρ - z_i) - y_i
dz_i/dt = x_i · y_i - β · z_i
```
观测变量：仅 x 分量

**Hénon 映射**：
```
X_i(t+1) = 1 - a · X_i(t)² + Y_i(t) + C · Σ_j A[i,j] · X_j(t)
Y_i(t+1) = b · X_i(t)
```
观测变量：仅 X 分量


### 6.3 替代数据方法 (`surrogate/`)

#### 统一接口

```python
from surrogate_ccm.surrogate import generate_surrogate

# 生成 100 条 iAAFT 替代数据
surrogates = generate_surrogate(x, method="iaaft", n_surrogates=100, seed=42)
# surrogates.shape = (100, T)
```

#### 各方法原理

| 方法 | 保持特征 | 破坏特征 | 计算速度 |
|------|---------|---------|---------|
| `fft` | 功率谱（自相关） | 非线性依赖、幅度分布 | 快 |
| `aaft` | 幅度分布 + 近似功率谱 | 非线性依赖 | 快 |
| `iaaft` | 幅度分布 + 功率谱（精确） | 非线性依赖 | 中等 |
| `timeshift` | 所有局部结构 | 耦合同步关系 | 最快 |
| `random_reorder` | 幅度分布 | 所有时间结构 | 最快 |

#### 单独调用各方法

```python
from surrogate_ccm.surrogate.fft_surrogate import fft_surrogate
from surrogate_ccm.surrogate.aaft_surrogate import aaft_surrogate
from surrogate_ccm.surrogate.iaaft_surrogate import iaaft_surrogate
from surrogate_ccm.surrogate.timeshift_surrogate import timeshift_surrogate
from surrogate_ccm.surrogate.random_reorder import random_reorder_surrogate

rng = np.random.default_rng(42)
surr = iaaft_surrogate(x, rng=rng, max_iter=200, tol=1e-8)
```

### 6.5 评估指标 (`evaluation/`)

```python
from surrogate_ccm.evaluation import evaluate_detection, compute_cohens_d, compute_delta_rho

# 检测性能评估
metrics = evaluate_detection(detected, ground_truth, scores=ccm_matrix)
```

**返回指标说明**：

| 指标 | 含义 | 理想值 |
|------|------|--------|
| TPR (True Positive Rate) | 真实因果边中被正确检测的比例 | → 1.0 |
| FPR (False Positive Rate) | 非因果边被错误检测的比例 | → 0.0 |
| precision | 检测结果中真正因果边的比例 | → 1.0 |
| F1 | Precision 和 Recall 的调和平均 | → 1.0 |
| AUC_ROC | ROC 曲线下面积（需提供连续分数） | → 1.0 |
| AUC_PR | PR 曲线下面积 | → 1.0 |
| TP / FP / FN / TN | 混淆矩阵各元素的计数 | — |

```python
# 效应量
d = compute_cohens_d(rho_obs, rho_surr)     # Cohen's d
delta = compute_delta_rho(rho_obs, rho_surr) # Δρ = ρ_obs - mean(ρ_surr)
```

## 9. 实验结果解读

### 9.1 热图解读

`plot_comparison_heatmaps` 生成四面板热图：

| 面板 | 内容 | 如何解读 |
|------|------|---------|
| CCM ρ | 观测 CCM 相关系数 | 颜色越深红 → CCM 值越高 → 因果证据越强 |
| p-values | 替代检验 p 值 | 颜色越绿 → p 值越小 → 越显著 |
| Detected | 二值检测结果 | 蓝色=检测到因果边，白色=未检测到 |
| Ground Truth | 真实邻接矩阵 | 用于对照检测结果 |

**注意**：矩阵中 `[i, j]` 位置表示 "j → i" 的检验结果。

### 9.2 网络图解读

`plot_network_comparison` 图中：

| 颜色 | 含义 | 代表 |
|------|------|------|
| 绿色实线 | True Positive | 正确检测到的因果边 |
| 红色实线 | False Positive | 误报的因果边 |
| 灰色虚线 | False Negative | 未检测到的真实因果边 |

### 9.3 性能曲线解读

- **TPR 随 ε 增大的典型趋势**：先上升后可能趋于平稳
  - ε < 0.05：耦合太弱，CCM 无法区分因果
  - ε = 0.05~0.2：最佳检测区间
  - ε > 0.3：强同步可能导致双向 CCM 值都高

- **FPR 随 ε 增大的典型趋势**：先低后可能升高
  - 强耦合导致广义同步，反向 CCM 也可能显著

### 9.4 收敛曲线解读

CCM 的核心验证特征是**收敛性**：

- **因果关系成立**：ρ 随 L（库大小）增大而单调递增并趋于稳定
- **无因果关系**：ρ 不随 L 明显增大，或波动不收敛
- **替代数据带**：观测 ρ 应显著高于替代数据的 95% 置信区间

## 10. 常见问题与排错

### Q1: Hénon 映射发散

```
RuntimeError: Henon map diverged at step XXX. Try reducing coupling=... or parameter a=...
```

**原因**：耦合过强或参数组合导致轨道逃逸。

**解决方案**：
- 降低耦合强度（如 0.3 → 0.1）
- 降低 Hénon 参数 a（如 1.4 → 1.2）
- Hénon 映射天然不稳定，建议耦合强度 ≤ 0.1

### Q2: Lorenz ODE 积分失败

```
RuntimeError: ODE integration failed: ...
```

**解决方案**：
- 减小 `dt`（如 0.01 → 0.005）
- 降低耦合强度
- 检查初始条件是否合理

### Q3: 运行太慢

**瓶颈分析**：
- N 节点网络 → N(N-1) 对因果检验 × n_surrogates 次替代计算
- 10 节点网络：90 对 × 100 替代 = 9000 次 CCM 计算

**加速方法**：
1. 减少 `n_surrogates`（100 → 30，快速测试）
2. 减少 `T`（3000 → 1500）
3. 减少 `n_reps`
4. 使用更快的替代方法（`timeshift` 或 `random_reorder`）
5. 增加 `--n-jobs` 并行度

### Q4: FPR 偏高

**可能原因**：
1. 耦合强度过大 → 广义同步导致虚假因果
2. 替代数据量不足 → p 值分辨率低
3. 时间序列太短 → CCM 未充分收敛
4. 未使用 FDR 校正 → 多重比较膨胀

**解决方案**：
1. 适当降低耦合强度
2. 增加 `n_surrogates`（至少 100，建议 200）
3. 增加时间序列长度 T
4. 确保使用 `fdr=True`

### Q5: TPR 偏低

**可能原因**：
1. 耦合强度太弱
2. 噪声太大
3. 嵌入参数不合适
4. 时间序列太短

**解决方案**：
1. 检查耦合强度是否在合理范围（0.05~0.3）
2. 降低观测噪声
3. 尝试手动设定嵌入参数
4. 增加 T

### Q6: 内存不足

对于大网络（N ≥ 30），替代数据矩阵可能很大：
- 100 替代 × 3000 时间步 × 30 节点 ≈ 每对需要 2.4 MB
- 总计 ~2 GB

**解决方案**：减少 `n_surrogates` 或 `T`，或分批运行。

## 11. 方法论背景

### 11.1 收敛交叉映射 (CCM)

CCM 由 Sugihara 等人 (2012, Science) 提出，基于 Takens 嵌入定理：

1. 对时间序列 x 进行延迟嵌入，重构影子流形 M_x
2. 如果 y 因果驱动了 x，则 M_x 中保留了 y 的信息
3. 可以从 M_x 的近邻关系交叉预测 y
4. **收敛性**是关键：随着库大小 L 增大，预测精度 ρ 应增大

### 11.2 替代数据检验

替代数据方法 (Theiler et al., 1992) 的核心思想：

1. **零假设**：观测到的 CCM 值可以由特定统计特征（如功率谱、幅度分布）完全解释
2. **替代数据**：保持零假设下的统计特征，但破坏因果结构
3. **检验**：如果观测 CCM 值显著大于替代数据分布 → 拒绝零假设 → 支持因果关系

### 11.3 统计检验流程

```
对每对 (i, j)：
  1. 计算观测 CCM：ρ_obs = ccm(x_i, x_j)  [检验 j→i]
  2. 生成 x_j 的 K 条替代数据：{x_j^(1), ..., x_j^(K)}
  3. 计算替代 CCM：{ρ^(1), ..., ρ^(K)}
  4. 秩 p 值：p = (#{ρ^(k) >= ρ_obs} + 1) / (K + 1)
  5. 收集所有 N(N-1) 个 p 值
  6. BH-FDR 校正
  7. p_corrected < α → 拒绝零假设 → 检测到 j→i
```

### 11.4 参考文献

- Sugihara, G., et al. (2012). Detecting Causality in Complex Ecosystems. *Science*, 338(6106), 496-500.
- Theiler, J., et al. (1992). Testing for nonlinearity in time series. *Physica D*, 58(1-4), 77-94.
- Schreiber, T., & Schmitz, A. (1996). Improved surrogate data for nonlinearity tests. *Physical Review Letters*, 77(4), 635.
- Benjamini, Y., & Hochberg, Y. (1995). Controlling the false discovery rate. *JRSS-B*, 57(1), 289-300.
